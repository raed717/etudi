# Teacher Dashboard — AI Context

## Overview

etudi is a Next.js 16 app for primary school teachers to share lessons with students. The teacher dashboard is the authenticated area where teachers manage their classes, groups, subjects, and student approvals.

**Stack**: Next.js 16.1.6 (App Router, Turbopack) / React 19 / TypeScript 5 / Tailwind v4 / Supabase (auth + Postgres) / pnpm

---

## Teaching Model

- One teacher teaches **multiple classes/grades** and **multiple subjects**
- Students self-register (role=`student`), enter a **6-char group join code**, teacher **approves/rejects**
- Subjects are a **fixed enum** of 10 values (mathematics, french, arabic, science, history, geography, civic_ed, art, pe, english)
- Academic year is **free text** (e.g. "2025-2026")
- Join codes are **auto-generated per group** by a DB trigger (BEFORE INSERT on `groups`), using chars `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (no I/O/0/1)

## Architecture Rules

1. **Server Components by default** — only add `"use client"` when needed (event handlers, useState, useTransition)
2. **CSS classes for hover effects** — never use JS `onMouseEnter`/`onMouseLeave`
3. **Form pattern**: React 19 `useActionState(serverAction, initialState)` for forms, `useTransition()` for imperative mutations (delete, approve/reject)
4. **Server Actions** (`"use server"`) handle all writes — validate, mutate DB, `revalidatePath()`, `redirect()`
5. **Auth guard is dual**: proxy middleware redirects unauthenticated users from `/dashboard*`, and each page also calls `getUser()` + `redirect("/sign-in")`
6. **Supabase clients**: `createClient()` from `@/lib/supabase/server` in Server Components/Actions, `createClient()` from `@/lib/supabase/client` in Client Components

## Database Schema

### Tables

```
profiles
  id              uuid PK (= auth.users.id)
  full_name       text
  avatar_url      text?
  role            user_role enum
  school          text?
  created_at      timestamptz
  updated_at      timestamptz

classes
  id              uuid PK
  name            text
  grade           int (1-6)
  academic_year   text
  teacher_id      uuid FK -> profiles.id
  created_at      timestamptz

class_subjects
  id              uuid PK
  class_id        uuid FK -> classes.id
  subject         subject enum

groups
  id              uuid PK
  class_id        uuid FK -> classes.id
  name            text
  join_code       char(6) — auto-generated by trigger, DEFAULT ''
  created_at      timestamptz

group_members
  id              uuid PK
  group_id        uuid FK -> groups.id
  student_id      uuid FK -> students.id
  status          membership_status enum (pending | approved | rejected)
  joined_at       timestamptz
  approved_at     timestamptz?

students
  id              uuid PK (= auth.users.id)
  full_name       text
  date_of_birth   date?
  created_at      timestamptz
```

### Enums

- `user_role`: teacher | admin | student
- `subject`: mathematics | french | arabic | science | history | geography | civic_ed | art | pe | english
- `membership_status`: pending | approved | rejected

### RLS Policies

All tables have RLS enabled. Policies use three `SECURITY DEFINER` helper functions to avoid infinite recursion:

- `is_class_teacher(class_id uuid)` — checks `classes.teacher_id = auth.uid()`
- `is_group_member(group_id uuid)` — checks `group_members.student_id = auth.uid()`
- `is_group_teacher(group_id uuid)` — joins groups->classes, checks `teacher_id = auth.uid()`

Key policies:
- `classes`: teacher CRUD on own rows (`teacher_id = auth.uid()`)
- `groups`: teacher CRUD via `is_class_teacher(class_id)`, student SELECT via `is_group_member(id)`
- `group_members`: teacher SELECT/UPDATE/DELETE via `is_group_teacher(group_id)`, student INSERT own (`student_id = auth.uid()`), student SELECT own
- `profiles` / `students`: SELECT/UPDATE own row only
- `students`: teacher reads via `is_group_teacher` on linked group_members

### Triggers

- `trg_groups_join_code` — BEFORE INSERT on `groups`, always fires, calls `generate_join_code()` which produces a unique 6-char alphanumeric code
- `handle_new_user()` — on auth.users INSERT: if role='student' inserts into `students`, otherwise inserts into `profiles`

## Route Map

```
(dashboard)/layout.tsx              — sidebar shell (SidebarServer + main area)

/dashboard                          — overview: greeting, 3 stat cards, quick links
/classes                            — class list grid (ClassCard components)
/classes/new                        — create class form (client component)
/classes/[classId]                  — class detail: subjects + groups (GroupCard)
/classes/[classId]/edit             — edit class form + delete danger zone
/classes/[classId]/groups/new       — create group form (client component)
/classes/[classId]/groups/[groupId] — group detail: join code, member list, approve/reject
/profile                            — teacher profile card
```

### Route params

Next.js 15+ async params pattern: `params` is `Promise<{ classId: string }>`, must `await params` before use.

## Server Actions (`src/server/actions/classes.ts`)

All return `ActionState = { error: string | null; success?: boolean }`.

| Action | Signature | Description |
|---|---|---|
| `createClass` | `(_prev, formData)` | Inserts class + optional subjects, redirects to class detail |
| `updateClass` | `(_prev, formData)` | Updates class name/grade/year by ID |
| `deleteClass` | `(classId)` | Deletes class, redirects to `/classes` |
| `addSubject` | `(classId, subject)` | Inserts into class_subjects (handles duplicate 23505) |
| `removeSubject` | `(classId, subject)` | Deletes from class_subjects |
| `createGroup` | `(_prev, formData)` | Inserts group (join_code omitted, trigger generates it) |
| `deleteGroup` | `(groupId, classId)` | Deletes group |
| `approveStudent` | `(memberId, groupId, classId)` | Sets status=approved, approved_at=now |
| `rejectStudent` | `(memberId, groupId, classId)` | Sets status=rejected |

Auth actions in `src/server/actions/auth.ts`: `signIn`, `signUp`, `signOut`.

## Data Flow

```
Browser request
  -> proxy.ts (refreshes Supabase session cookie)
  -> (dashboard)/layout.tsx
       -> SidebarServer (fetches user, profile, pending count)
            -> Sidebar (client: nav, collapse, sign-out)
       -> Page (Server Component: fetches page-specific data)
            -> Client Components (forms, buttons via useActionState/useTransition)
                 -> Server Actions (validate, DB write, revalidatePath, redirect)
```

## Middleware (`src/proxy.ts`)

Next.js 16 renamed `middleware.ts` to `proxy.ts`, exported function must be named `proxy`.

- Refreshes Supabase auth session on every request
- Redirects unauthenticated users from `/dashboard*` to `/sign-in`
- Redirects authenticated users from `/sign-in`, `/sign-up` to `/dashboard`
- Note: `/classes/*` and `/profile` routes are NOT protected by middleware (only by in-page `getUser()` checks)

## Known Patterns & Gotchas

1. **Both `/app/page.tsx` and `/app/(marketing)/page.tsx` cannot coexist** — they both resolve to `/`. Keep content in `/app/page.tsx`, leave `(marketing)/page.tsx` as a stub.
2. **CSS `@import` ordering** — Google Fonts imports must come before `@import "tailwindcss"` in globals.css (Tailwind v4 enforces this).
3. **N+1 queries** — classes list and class detail pages fetch counts per-item in `Promise.all`. Not a problem at current scale but worth noting.
4. **`SUBJECT_LABELS` / `SUBJECT_COLORS` are duplicated** — defined in both `class-card.tsx` and `classes/[classId]/page.tsx`. Consider extracting to a shared constant.
5. **`src/types/index.ts` is stale** — defines `Role` as `"instructor"` which diverges from the DB enum `"teacher"`. Not used by dashboard code.
6. **LSP ghost error** — may show error for deleted `src/middleware.ts`. Ignore it.
7. **`src/server/db/index.ts`** is a placeholder (`db = null`). All DB access uses the Supabase client directly.
